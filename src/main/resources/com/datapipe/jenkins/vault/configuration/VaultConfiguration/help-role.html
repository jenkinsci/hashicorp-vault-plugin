Specify token role name to use when creating new tokens (see Create / update token role). Roles enforce specific behavior when creating tokens that allow token functionality that is otherwise not available or would require sudo/root privileges to access. Role parameters, when set, override any provided options to the create endpoints. The role name is also included in the token path, allowing all tokens created against a role to be revoked using the /sys/leases/revoke-prefix endpoint. Token roles are different from AppRole roles and also need to exist in order to use them.
